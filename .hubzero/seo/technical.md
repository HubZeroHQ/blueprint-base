# Technical SEO

## Metadata
- Every page must have a unique title.
- Every page must have a unique meta description.
- Titles should accurately describe the page.
- Avoid duplicate metadata.

## Canonical URLs
- Every indexable page should define a canonical URL.
- Canonicals should resolve duplicate content.

## Open Graph
- Every shareable page should define Open Graph metadata.
- Images should meet platform recommendations.

## Twitter Cards
- Configure Twitter/X metadata consistently.

## Structured Data
- Use appropriate schema types.
- Never publish false structured data.
- Prefer complete structured data over partial implementations.

## Sitemap
- Automatically generate XML sitemaps.
- Include only canonical indexable URLs.
- Exclude duplicate, filtered, or private pages.

## Robots
- Prevent indexing of private or administrative areas.
- Ensure production sites expose crawlable public content.

## URL Structure
- Stable URLs.
- Human-readable.
- Lowercase.
- Hyphen-separated.
- Avoid unnecessary nesting.

## Heading Structure
- One primary heading.
- Logical hierarchy.
- Do not skip heading levels without reason.

## Images
- Descriptive filenames.
- Meaningful alt text.
- Responsive sizing.
- Modern formats where appropriate.

## Performance
- Optimize Core Web Vitals.
- Avoid unnecessary JavaScript.
- Optimize fonts and images.
- Reduce layout shifts.

## Accessibility
- Semantic HTML.
- Descriptive links.
- Keyboard support.
- Correct landmarks.

## Crawlability
- Public pages should be discoverable through navigation.
- Avoid orphan pages.
- Internal links should reinforce site structure.

## Internationalization
- Use hreflang where multilingual content exists.
- Avoid duplicate translated content.

## Monitoring
- Validate structured data.
- Monitor crawl errors.
- Monitor indexing.
- Review broken links.
- Periodically review metadata quality.
